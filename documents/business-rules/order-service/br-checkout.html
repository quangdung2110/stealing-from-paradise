BR-ORDER-001 to BR-ORDER-009: Checkout Business Rules

Stable IDs: BR-ORDER-001 through BR-ORDER-009 Domain: Checkout via Product Service (POST /v1/cart/checkout/preview, POST /v1/cart/checkout/submit) Last Updated: 2026-06-09

BR-ORDER-001: Cart Selection

Rule: Checkout preview requires 1 to 50 selected cart item IDs.

Condition Outcome
IF item_ids is empty Reject with 400
IF item_ids has more than 50 items Reject with 400
IF all selected cart items are valid Continue to preview validation

BR-ORDER-002: Preview Validation

Rule: Product Service validates selected variants before issuing a preview_token.

Check Required Outcome
Variant exists and is active Must pass
Price snapshot matches current price Must pass
Stock is sufficient Must pass
Cart item belongs to buyer Must pass

Token: preview_token is stored in Redis and expires after 10 minutes.

BR-ORDER-003: Address Validation

Rule: Product Service validates the buyer address with Identity Service before checkout submit continues.

Step Action
1 Product Service sends order.address.request with user_id and address_id
2 Identity Service replies on order.address.response
3 IF address is missing or does not belong to user, Product Service rejects with 409
4 IF valid, Product Service uses the Identity response to build address_snapshot

BR-ORDER-004: Stock Reservation

Rule: Checkout submit revalidates stock and creates pending reservations before order creation.

Condition Outcome
IF stock or price changed after preview Reject with 409
IF stock is available Create PENDING reservations with the checkout session_id

Reservation TTL: 15 minutes.

BR-ORDER-005: Checkout Submit Event

Rule: Product Service emits order.checkout_submitted after address validation and stock reservation.

Field Source
session_id Generated by Product Service
customer_id Authenticated buyer
address_id Submit request
address_snapshot Identity Service response
items[] Product/cart snapshots
total_amount, total_items Calculated from selected items

BR-ORDER-006: Order Creation from Event

Rule: Order Service creates the parent order and seller sub-orders only from order.checkout_submitted.

Step Action
1 Consume order.checkout_submitted
2 Group items by seller_id
3 Create one PARENT_ORDERS row
4 Create one ORDERS row per seller
5 Create ORDER_ITEMS snapshots
6 Store session_id on the parent order

Atomicity: Parent, orders, and order items are created in one database transaction.

BR-ORDER-007: Order Code Generation

Rule: Each sub-order receives a unique human-readable order_code.

Format: OR-YYYYMMDD-{orders.id}

BR-ORDER-008: Parent-Order Financial Integrity

Rule: Parent order amounts must equal the sum of sub-orders.

Condition Outcome
IF parent_order.final_amt != SUM(orders.final_amt) Abort transaction

BR-ORDER-009: Events After Order Creation

Rule: Order Service publishes Axon events after creating orders.

Event Effect
OrderCreatedEvent Starts per-order saga and publishes Kafka order.created
ParentOrderCheckoutCreatedEvent Starts parent payment saga and publishes Kafka payment.requested

Error Response Summary

HTTP / Async Error Condition
400 Invalid checkout preview/submit request
409 Preview token invalid, stock changed, or address invalid
501 Deprecated Order Service POST /orders/checkout called directly
Internal retry/log Invalid order.checkout_submitted payload
